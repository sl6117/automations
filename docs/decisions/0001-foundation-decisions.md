# 0001 — Foundation decisions

Date: 2026-06-10

Decisions made while building the personal automation foundation + Twitter digest (project #1).

## Working mode
Guided driver: the owner runs/writes and learns; the assistant explains and reviews. Code is
built to be inspectable and runnable end-to-end offline.

## Runtime: local-first, portable
Runs on the Mac to start ($0 hosting, fastest iteration). Everything is written to lift-and-shift
to a VPS later (no Mac-specific assumptions except the launchd scheduler, which a VPS would
replace with system cron/systemd).

## Data source: bird now, swappable later
`lib/fetch/` is an interface. `bird` (cookie-based X CLI) is the first adapter. A `mock` adapter
runs the whole pipeline offline. Xquik (official API, paid) can be added as a third adapter via
`FETCH_SOURCE=xquik` with zero changes to the digest logic. Rationale: free to learn now, no
account-ban risk later when reliability matters.

## AI efficiency
- Filter before the model: engagement + dedup + incremental cursor run in `filter.js` (no tokens).
  Only the top N already-deduped tweets reach the LLM.
- Cheapest capable model: default `anthropic/claude-haiku-4.5`. Escalate per-task if needed.
- Slim prompt: only author/handle/url/text/score are sent, not raw API payloads.
- Context on demand: `AGENTS.md` stays small; `references/` is documentation, not sent to the model.
- Dry-run path: pipeline produces a heuristic digest with no API call for testing.

## Cost logging
Every real LLM run appends to `logs/cost-log.jsonl`; `npm run cost:report` summarizes spend.

## Security (supply chain)
- Pre-download audit script `scripts/npm-preaudit.sh` vets any npm package before install
  (metadata, publish-freshness warning, full-tree `npm audit`) without running install scripts.
- Device-wide npm/npx download log at `~/Library/Logs/npm-downloads.log` via a zsh hook in
  `~/.npm-audit/npm-logger.zsh`. See `docs/security/`.
- Session installs audited 2026-06-10: Node + direnv (official, verified); openclaw (clean, one
  moderate transitive `hono` issue, loopback-only); bird (clean). See `docs/security/audit-2026-06-10.md`.

## Scheduling: launchd, NOT in ~/Desktop  (important)
macOS TCC blocks background jobs (launchd) from reading `~/Desktop`, `~/Documents`, `~/Downloads`
("Operation not permitted", exit 126). Confirmed via a control test (a `/tmp` job ran fine).
Therefore the project lives at `~/automations/twitter_summary`, and the 9am digest is a launchd
LaunchAgent generated by `scripts/schedule-launchd.sh`. The agent appears in System Settings as a
`bash` "background item from unidentified developer" — this is normal for a hand-authored
LaunchAgent and is the legitimate digest scheduler.

## Delivery
Direct Telegram Bot API (`lib/deliver/telegram.js`), independent of OpenClaw, so the pipeline is
self-contained and testable. OpenClaw remains the optional agent/chat + alternative scheduler.
